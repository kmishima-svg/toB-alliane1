# 法人管理アプリ セットアップ手順

## 1. GitHub Pagesへの公開

1. GitHubで新しいリポジトリを作成(例: `corp-crm`)
2. `index.html` をリポジトリの**ルート**に配置してpush
   ```
   git init
   git add index.html
   git commit -m "初版"
   git remote add origin https://github.com/<ユーザー名>/corp-crm.git
   git push -u origin main
   ```
3. リポジトリの Settings → Pages → Source を「main / root」に設定
4. 数分後、`https://<ユーザー名>.github.io/corp-crm/` でアクセス可能になります
   - このURLを以降の手順で使用します(以下「公開URL」と表記)

## 2. Anthropic APIキーの取得

1. https://console.anthropic.com にログイン
2. 左メニュー「API Keys」→「Create Key」
3. 発行されたキー(`sk-ant-...`)をコピー
4. アプリの「設定・連携」タブに貼り付けて保存
   - キーはブラウザのlocalStorageにのみ保存され、外部サーバーには送信されません
   - 利用料金はAnthropic APIの通常料金(従量課金)が発生します

## 3. Google Cloud OAuthクライアントIDの取得

1. https://console.cloud.google.com で新規プロジェクトを作成
2. 「APIとサービス」→「ライブラリ」から以下を有効化
   - **Google Calendar API**
   - **Google Sheets API**
3. 「APIとサービス」→「OAuth同意画面」
   - User Type: 外部(External)
   - アプリ名・サポートメールなど必須項目を入力
   - スコープ: `calendar`、`spreadsheets`、`drive.file` を追加
   - テストユーザーに自分のGoogleアカウントを追加(公開審査前でも自分は使えます)
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
   - アプリケーションの種類: **ウェブアプリケーション**
   - 「承認済みのJavaScript生成元」に**手順1の公開URL**を追加
     例: `https://yourname.github.io`(末尾のパスやスラッシュは付けない)
   - リダイレクトURIの設定は不要(このアプリはトークンクライアント方式を使用)
5. 発行された「クライアントID」(`xxxx.apps.googleusercontent.com`)をコピー
6. アプリの「設定・連携」タブに貼り付けて保存し、「サインイン」ボタンでGoogle認証

> 注意: OAuth同意画面が「テスト」状態の場合、トークンは1週間程度で失効します。継続利用する場合は「本番公開」への切替(Googleの審査が必要な場合あり)をご検討ください。社内利用のみであれば「テスト」のままテストユーザーを追加して使う運用でも問題ありません。

## 4. スプレッドシート同期

- 設定タブの「新規シート作成」で、3シート構成(企業一覧/担当者/MTG記録)の新規スプレッドシートを自動作成できます
- 既存のシートを使う場合は、スプレッドシートのURLから次の形式でIDを取り出して入力してください
  ```
  https://docs.google.com/spreadsheets/d/【ここがID】/edit
  ```
  その場合、シート名が「企業一覧」「担当者」「MTG記録」と完全一致するタブを事前に作成しておいてください
- 「今すぐ同期」を押すたびに、各シートの内容を全件上書き更新します

## 5. 使い方の流れ

1. 「企業を追加」または「名刺スキャン」から企業・担当者を登録
2. 企業カードを開き「Web検索で企業情報を取得」→ 内容を確認(自動取得情報は誤りを含む可能性があるため必ず目視確認してください)
3. 「MTG記録」タブでMTG内容・次回アクションを入力し「保存してカレンダー登録」
4. 「設定・連携」タブの「今すぐ同期」でスプレッドシートに反映

## データの保存場所について

- すべてのデータ(企業・担当者・MTG記録・APIキー)はこの端末の**ブラウザのlocalStorage**にのみ保存されます
- 別の端末・ブラウザでは別データになります(共有が必要な場合はスプレッドシート同期をデータの正本として運用してください)
- ブラウザのキャッシュ削除やシークレットモードでは保存内容が消える場合があります

## 既知の制約

- Web検索による企業情報取得は、検索結果の精度に依存します。重要な意思決定に使う前は必ず正式な情報源(有報・会社HP等)で確認してください
- 名刺の手書き文字や特殊なレイアウトは読み取り精度が下がることがあります
- OAuth同意画面が「テスト」モードの間は、登録したテストユーザー以外はログインできません
